헤더 (필수)
형식:

<타입>(<스코프>): <간단한 설명> (#<이슈번호>)


<타입>: 변경의 성격을 나타냄
feat: 새로운 기능 추가
fix: 버그 수정
docs: 문서 변경
refactor: 기능 변화 없이 코드 구조 변경
test: 테스트 관련 추가/수정
build: 빌드/배포 관련 설정 변경
ci: CI 환경 관련 변경
perf: 성능 개선

<스코프>: 변경된 영역 (선택)
예: api, ui, db, auth 등. 필요 없으면 생략 가능.

<간단한 설명>:
현재 시제 사용
문장 끝에 마침표(.) 쓰지 않음

<이슈번호> :
개인 repo : 기능 구현 전 작성한 Issue번호 추가
통합 repo : PR 번호 추가


예시:
feat(api): 사용자 로그인 엔드포인트 추가 (#2)
fix(ui): 버튼 정렬 수정 (#3232)
docs: README 업데이트 (#634)


본문 (선택, 가능하면 작성)
변경 이유, 상세 설명, 이전 동작과 새로운 동작의 차이 등을 적음
20자 이상, 여러 줄 가능

로그인 엔드포인트를 추가해 사용자 인증 기능 구현.
